## 1. 接口概述

Gin扩展框架提供了一系列丰富的HTTP API接口，旨在简化Web开发流程，提高开发效率。本文档详细描述了框架所提供的核心API接口，包括请求格式、响应结构和使用示例。

## 2. 通用说明

### 2.1 请求格式

#### HTTP方法使用规范

| HTTP方法 | 使用场景 | 示例 |
|---------|---------|------|
| GET | 数据查询，不修改资源 | 获取用户信息、查询列表 |
| POST | 创建新资源 | 创建用户、提交表单 |
| PUT | 更新资源（完整替换） | 更新用户全部信息 |
| PATCH | 部分更新资源 | 更新用户部分字段 |
| DELETE | 删除资源 | 删除用户、取消订单 |

#### 请求头要求

```
Content-Type: application/json
Authorization: Bearer {token}  (需要认证的接口)
Accept-Language: zh-CN  (可选，用于国际化)
X-Request-ID: {requestId}  (可选，用于请求跟踪)
```

#### 请求参数类型

- 路径参数：`/api/users/{id}`
- 查询参数：`/api/users?page=1&size=10`
- 请求体参数：JSON格式的请求体

### 2.2 响应格式

所有API接口的响应均采用统一的JSON格式：

```json
{
  "code": 0,
  "message": "操作成功",
  "data": {
    // 响应数据，根据接口不同而变化
  },
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

#### 字段说明

| 字段名 | 类型 | 说明 |
|-------|------|------|
| code | int | 状态码，0表示成功，非0表示错误 |
| message | string | 响应消息，成功或错误描述 |
| data | object/array | 响应数据，可能是对象或数组 |
| timestamp | int64 | 响应时间戳（Unix秒级） |
| requestId | string | 请求唯一标识符 |

#### 分页响应格式

对于分页查询接口，data字段使用以下格式：

```json
{
  "code": 0,
  "message": "查询成功",
  "data": {
    "items": [
      // 数据项数组
    ],
    "pagination": {
      "page": 1,
      "size": 10,
      "total": 100,
      "pages": 10
    }
  },
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

### 2.3 错误码说明

| 错误码 | 说明 | HTTP状态码 |
|-------|------|-----------|
| 0 | 成功 | 200 |
| 1000 - 1999 | 参数错误 | 400 |
| 2000 - 2999 | 认证授权错误 | 401/403 |
| 3000 - 3999 | 业务逻辑错误 | 400 |
| 4000 - 4999 | 资源不存在 | 404 |
| 5000 - 5999 | 服务器内部错误 | 500 |

## 3. 认证授权接口

### 3.1 用户登录

**接口描述**：用户通过用户名和密码进行登录，获取访问令牌

**请求方法**：POST

**请求路径**：`/api/auth/login`

**请求参数**：

```json
{
  "username": "string",
  "password": "string",
  "rememberMe": false
}
```

**响应示例**：

```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 86400,
    "userInfo": {
      "id": 1,
      "username": "admin",
      "role": "admin"
    }
  },
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

**错误码**：

| 错误码 | 说明 | HTTP状态码 |
|-------|------|-----------|
| 2001 | 用户名或密码错误 | 401 |
| 2002 | 账户已被锁定 | 403 |

### 3.2 刷新令牌

**接口描述**：使用现有令牌获取新的访问令牌

**请求方法**：POST

**请求路径**：`/api/auth/refresh`

**请求头**：

```
Authorization: Bearer {token}
```

**响应示例**：

```json
{
  "code": 0,
  "message": "刷新成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 86400
  },
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

**错误码**：

| 错误码 | 说明 | HTTP状态码 |
|-------|------|-----------|
| 2003 | 无效的令牌 | 401 |
| 2004 | 令牌已过期 | 401 |

### 3.3 用户注销

**接口描述**：用户注销当前会话

**请求方法**：POST

**请求路径**：`/api/auth/logout`

**请求头**：

```
Authorization: Bearer {token}
```

**响应示例**：

```json
{
  "code": 0,
  "message": "注销成功",
  "data": null,
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

## 6. 事件流接口

### 6.1 建立SSE连接

**接口描述**：建立服务器发送事件(SSE)连接，接收实时事件推送

**请求方法**：GET

**请求路径**：`/api/events/stream`

**请求头**：

```
Authorization: Bearer {token}
```

**响应内容类型**：`text/event-stream`

**事件格式**：

```
event: message
data: {"type": "notification", "content": "新消息通知"}

event: update
data: {"type": "status", "entityId": 123, "status": "completed"}
```

**事件类型说明**：

| 事件类型 | 说明 | 数据格式 |
|---------|------|---------|
| message | 通用消息通知 | `{"type": "string", "content": "string"}` |
| update | 状态更新通知 | `{"type": "string", "entityId": number, "status": "string"}` |
| error | 错误通知 | `{"code": number, "message": "string"}` |

## 7. 文件操作接口

### 7.1 文件上传

**接口描述**：上传文件

**请求方法**：POST

**请求路径**：`/api/files/upload`

**请求头**：

```
Content-Type: multipart/form-data
Authorization: Bearer {token}
```

**请求参数**：

| 参数名 | 类型 | 位置 | 必填 | 说明 |
|-------|------|------|-----|------|
| file | file | form-data | 是 | 要上传的文件 |
| type | string | form-data | 否 | 文件类型 |
| description | string | form-data | 否 | 文件描述 |

**响应示例**：

```json
{
  "code": 0,
  "message": "文件上传成功",
  "data": {
    "fileId": "f12345",
    "fileName": "example.pdf",
    "fileSize": 1024000,
    "fileUrl": "/files/f12345",
    "uploadTime": "2021-07-01T12:00:00Z"
  },
  "timestamp": 1625123456,
  "requestId": "req-123456"
}
```

**错误码**：

| 错误码 | 说明 | HTTP状态码 |
|-------|------|-----------|
| 1101 | 文件大小超过限制 | 400 |
| 1102 | 不支持的文件类型 | 400 |
| 5101 | 文件存储失败 | 500 |

## 8. 安全与性能建议

### 8.1 接口安全建议

1. **认证与授权**：
   - 敏感接口必须进行身份验证和授权检查
   - 使用HTTPS确保传输层安全
   - JWT令牌存储在安全位置，避免XSS攻击

2. **输入验证**：
   - 所有用户输入必须进行严格验证
   - 使用参数绑定和验证标签进行自动验证
   - 防范SQL注入、XSS等常见攻击

3. **限流与防护**：
   - 为公开API实施请求速率限制
   - 防范DDoS和暴力破解攻击
   - 使用验证码或其他挑战机制防止自动化攻击

### 8.2 性能优化建议

1. **响应优化**：
   - 使用适当的缓存策略
   - 分页查询大数据集
   - 仅返回必要的字段数据

2. **连接管理**：
   - 使用连接池管理数据库连接
   - 及时关闭不需要的资源
   - 限制SSE连接的最大数量

3. **异步处理**：
   - 对耗时操作使用异步处理
   - 文件处理等耗时任务应在后台进行
   - 使用消息队列分发任务
